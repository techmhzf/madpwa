# PWA Example: Twitter

## Aim
To study Web Applications, Native Applications, and Progressive Web Applications (PWA) and understand their features, advantages, and differences.

---

## Software Required
- Flutter SDK  
- Android Studio / VS Code  
- Android Emulator or Physical Device  

---

## Theory

### 1. Web Applications (Web Apps)
A web application is an application that is hosted on a web server and accessed through a web browser via the Internet or an intranet. Unlike native applications, web applications do not need to be installed on the user’s device.

**Examples:** Gmail, Facebook, Spotify, Netflix

---

### 2. Native Applications
A native application is a software application developed specifically for a particular operating system such as Android, iOS, or Windows. These applications are installed directly on the device from application stores like Google Play Store or Apple App Store.

**Examples:** Facebook app, WhatsApp, Instagram, Snapchat

---

## Questions and Answers

### Q1. What is a Web Application?
A web application is a software program that runs on a web server and is accessed through a web browser.

**How it works:**
- User sends request via browser
- Server processes request
- Server sends response back to browser

**Examples:** Gmail, Netflix

---

### Q2. Advantages of Web Applications
- No installation required  
- Accessible from any device with internet  
- Easy to update and maintain  
- Platform independent  

---

### Q3. What is a Native Application?
A native application is built specifically for a particular operating system and installed on the device.

**Difference from Web Apps:**
- Requires installation  
- Works faster and smoother  
- Can access device hardware  

---

### Q4. Advantages of Native Applications
- High performance  
- Better user experience  
- Works offline  
- Access to device features (camera, GPS, etc.)  

---

### Q5. What is a Progressive Web Application (PWA)?
A Progressive Web Application (PWA) is a type of web application that combines features of both web and native applications.

**Key Features:**
- Works in browser like web apps  
- Can be installed like native apps  
- Works offline  
- Fast and reliable  

---

### Q6. Architecture of a PWA

**Components:**

- **Service Worker:**  
  A background script that handles caching and offline functionality.

- **Server:**  
  Stores and delivers web content.

- **Cache Storage:**  
  Stores resources locally for faster loading and offline use.

- **Online & Offline Operation:**  
  Works with internet and also without internet using cached data.

---

### Q7. What is a Service Worker?
A Service Worker is a script that runs in the background and helps in caching data, enabling offline access, and improving performance.

**Importance:**
- Enables offline functionality  
- Improves speed  
- Supports push notifications  

---

### Q8. Practical Observation

#### Steps:
1. Open a web browser on your computer or mobile device.  
2. Visit any application (example: Twitter, Flipkart, etc.).  
3. Check whether the application:
   - Works directly in the browser  
   - Can be installed on the device  
   - Works offline  
4. Try installing a PWA from the browser if available.  
5. Observe behavior with and without internet connection.  

#### Observation:
- Web apps run directly in browser  
- PWA shows install option  
- PWA works partially offline  
- Native apps require installation  

#### Conclusion:
Different types of applications provide different levels of performance, accessibility, and offline capability. PWAs combine the benefits of both web and native applications.
